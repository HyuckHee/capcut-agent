# 영화/드라마 짤 — 기승전결 알고리즘

사용자 피드백으로 확립. 짤 만들 때 **매번 이 순서를 따른다.** 상상으로 스토리를 지어내지 말 것.

## 0. 소스 완전 파악 (스토리 짜기 전 필수)
- `prep_owl.py`류로 대상 클립을 **ASCII 경로로 복사** (한글 경로는 ffmpeg가 못 읽음 — PowerShell↔native exe 경계에서 깨짐).
- `frames.py <영상> <ss> <to> <fps> <cols> <rows> <out>`로 **전 구간을 1~2fps 몽타주**로 실제 확인.
- `python -m app.transcribe`로 전체 대본. (중반 저대사 구간 타임스탬프는 부정확 → 정제)
- 파일명 타임스탬프로 촬영 연속성 확인. **같은 씬이면 클립 하나만 사용** (여러 개 섞으면 뒤죽박죽 — 사용자 지적).

## 1. 반전(TWIST) 먼저 특정
이 영상의 **핵심 반전/하이라이트**가 뭔지 한 문장으로 적는다.
예) 올빼미: "맹인인 줄 알았던 침술사가 사실 야맹증 → 불이 꺼지자 독살을 목격."
반전을 못 찾으면 짤로 만들 가치가 없다. 반전이 곧 후킹 포인트.

## 2. 기승전결 배치 (반전 중심으로 역설계)
- **起(Hook, ~0-8s)**: 상황 + 긴장 질문. "무슨 일이 벌어지는가?" (세자가 죽어간다)
- **承(Build)**: 인물/상황 전개 + **반전의 복선을 반드시 여기서 심는다.**
  복선은 반전보다 먼저. (그는 맹인으로 알려졌다 / 사실 어둠에서 본다는 비밀)
- **轉(Twist)**: 반전을 터뜨린다. 시각적 전환점(불 꺼짐)에 맞춰 설명자막+나레이션으로 명시.
- **結(Payoff)**: 감정적 마무리 + 여운/아이러니 + 다음 궁금증. (유일한 목격자가 맹인이라는 비극)

## 3. 시간 순서 지키되 압축
- 원본의 시간 순서를 유지 (관객이 인과를 따라가야 함).
- 지루한 구간은 건너뛰되, 인과에 필요한 샷은 남긴다.
- 피사체가 프레임 밖으로 나가거나 흔들리는 꼬리 컷은 버린다.

## 4. 나레이션/설명자막이 맥락을 전달
- 영상만으론 관객이 못 잡는 배경(복선·반전·아이러니)을 **말과 글로 명시**.
- 대사자막(흰색 작게 하단) = 실제 대사. 설명자막(노란 박스 크게) = 나레이터 해설.
- 나레이션(edge-tts) = 복선·반전·마무리 3박자. 과하지 않게 (54s에 3~4개).

## 5. 렌더 후 검증
- 프레임 뽑아 **자막이 그 순간 화면 내용과 맞는지** 확인.
- 반전 지점(불 꺼짐)과 나레이션·자막 싱크 확인.

## 도구
- `render_cinema.py`: `--seg`(파일\|A\|B 또는 A-B), `--crop`, `--title "줄1\|줄2"`,
  `--sub "A-B:대사"`, `--exp "A-B:설명"`, `--narr "A:나레이션"`,
  `--voice`/`--narr-rate`/`--narr-pitch`.
- 한글 경로 회피: 입력은 ASCII 복사본, 출력도 ASCII로 뽑은 뒤 Python(shutil)으로 최종 폴더 이동.
